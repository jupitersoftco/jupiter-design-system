# Contributing to Jupiter Design System

Thank you for your interest in contributing to the Jupiter Design System!

## How to Contribute

1. **Fork the repository** and create a new branch for your changes
2. **Make your changes** following the existing code style and patterns
3. **Ensure all tests pass** by running `cargo test`
4. **Open a Pull Request** with a clear description of your changes

## Requirements

- All tests must pass before your PR will be reviewed
- Code should follow Rust best practices and existing project conventions
- Include tests for new functionality when applicable

## Review Process

Once you submit a PR that meets the above requirements, I will review it and provide feedback or merge it if approved.

## Questions?

For questions about the project or contribution process, visit [jupitersoft.net](https://jupitersoft.net) or open an issue.
